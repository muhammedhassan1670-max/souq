ملخص تطوير لوحة أدمن سوق البلد
================================

الغرض من الملف
---------------
هذا الملف يشرح ما تم تنفيذه في لوحة الأدمن حتى الآن، بحيث يمكن إرساله لأي أداة أو مطور آخر للبناء عليه بدون الرجوع لكل تفاصيل المشروع.


1. تشغيل حقيقي بالأدمن وقاعدة البيانات
--------------------------------------
- تم ربط المشروع بـ Supabase كـ Backend وقاعدة بيانات.
- تم إنشاء ملف src/lib/supabase.ts للتعامل مع مفاتيح Supabase من البيئة.
- تم إنشاء خدمات منفصلة للبيانات:
  - src/services/productsService.ts
  - src/services/categoriesService.ts
  - src/services/shopsService.ts
  - src/services/ordersService.ts
  - src/services/settingsService.ts
  - src/services/uploadService.ts
  - src/services/authService.ts
- تم إنشاء schema في supabase/schema.sql يحتوي على جداول:
  - categories
  - shops
  - products
  - orders
  - order_items
  - settings
  - custom_requests
  - seller_requests
- تم تفعيل RLS بحيث العميل يقرأ البيانات العامة فقط، والأدمن يدير البيانات بعد تسجيل الدخول.


2. تسجيل دخول الأدمن
--------------------
- تم إنشاء تسجيل دخول حقيقي باستخدام Supabase Auth.
- المسارات الخاصة بالأدمن:
  - /admin/login
  - /admin
  - /admin/products
  - /admin/categories
  - /admin/shops
  - /admin/orders
  - /admin/settings
- تم حماية مسارات الأدمن عبر ProtectedAdminRoute.
- غير المسجل يتم تحويله إلى /admin/login.
- تم إخفاء أي زر دخول أدمن من واجهة العميل، والدخول يكون من الرابط المباشر فقط.


3. هيكل لوحة الأدمن
-------------------
- تم إنشاء AdminShell يحتوي على:
  - لوجو سوق البلد
  - Navigation للأقسام الأساسية
  - زر خروج
  - تصميم RTL عربي
- Dashboard يعرض أرقام مختصرة عن:
  - المنتجات
  - العروض
  - المحلات
  - الطلبات


4. إدارة المنتجات
-----------------
الملف الأساسي: src/pages/admin/AdminProducts.tsx

الموجود حاليًا:
- عرض المنتجات من Supabase.
- بحث باسم المنتج أو القسم أو كلمات البحث.
- فلتر حسب القسم.
- إحصائيات سريعة:
  - كل المنتجات
  - العروض
  - غير متاح
  - مميز
- إضافة منتج جديد.
- تعديل منتج كامل.
- حذف منتج مع تأكيد.
- رفع صورة المنتج إلى Supabase Storage.
- Preview للصورة قبل الحفظ.
- تغيير:
  - اسم المنتج
  - الوصف
  - القسم
  - المحل
  - السعر
  - السعر القديم
  - الوحدة
  - الصورة
  - التوفر
  - الكمية
  - عرض
  - منتج بلدي
  - منتج مميز
  - كلمات البحث
  - Tags
  - ترتيب الظهور

تطوير حديث لتسهيل تحديث الأسعار:
- تمت إضافة صندوق "تحديث سريع للسعر" داخل كل كارت منتج.
- الأدمن يستطيع تعديل هذه القيم مباشرة بدون فتح فورم المنتج الكامل:
  - السعر الجديد
  - السعر القديم
  - الكمية
  - متاح / غير متاح
  - عرض / ليس عرض
- زر "حفظ السعر" يحدث المنتج فورًا في Supabase.
- بعد الحفظ يتم عمل refresh وتظهر التغييرات للعميل.
- يوجد validation بسيط:
  - السعر مطلوب
  - السعر لا يكون بالسالب
  - السعر القديم لا يكون بالسالب
  - الكمية لا تكون بالسالب


5. إدارة الأقسام
----------------
الملف الأساسي: src/pages/admin/AdminCategories.tsx

الموجود حاليًا:
- عرض كل الأقسام.
- إضافة قسم جديد.
- تعديل قسم.
- حذف قسم مع تأكيد.
- رفع صورة القسم.
- تغيير:
  - اسم القسم
  - Slug
  - Icon
  - ترتيب الظهور
  - الصورة
  - ظاهر للعميل / مخفي

تطوير حديث لإدارة حالة القسم:
- تمت إضافة خيار "قفل القسم مؤقتًا".
- عند تفعيل الخيار:
  - القسم يظل ظاهرًا للعميل.
  - يظهر عليه Badge باسم "قريبًا".
  - منتجات القسم لا تظهر للعميل.
  - إذا دخل العميل القسم تظهر رسالة أن القسم قريبًا.
- تم إضافة بحث داخل صفحة الأقسام.
- تم إضافة فلتر حالة:
  - الكل
  - مفتوح
  - قريبًا
  - مخفي
- تم إضافة إحصائيات:
  - كل الأقسام
  - مفتوح
  - قريبًا
  - مخفي
- تم إضافة أزرار سريعة من كارت القسم:
  - إظهار / إخفاء
  - قريبًا / فتح القسم

تغيير قاعدة البيانات المطلوب:
- تم إضافة عمود coming_soon إلى جدول categories.
- يجب تشغيل supabase/schema.sql على Supabase الحقيقي لتفعيل العمود والسياسات الجديدة.


6. تأثير الأقسام على واجهة العميل
---------------------------------
- القسم المخفي active=false لا يظهر للعميل.
- القسم coming_soon=true يظهر للعميل لكن بعلامة "قريبًا".
- منتجات القسم coming_soon لا تظهر في:
  - Home
  - Products
  - Offers
  - LocalProducts
- صفحة Products تعرض Empty State مناسب إذا اختار العميل قسمًا قريبًا.


7. الطلبات
-----------
الملف الأساسي: src/pages/admin/AdminOrders.tsx

الموجود حاليًا:
- عرض الطلبات المحفوظة من Supabase.
- حفظ الطلبات القادمة من checkout.
- حفظ order_items.
- رقم طلب من شكل SB-YYYY-0001.
- البحث برقم الطلب أو رقم العميل.
- فلترة حسب الحالة.
- تحديث حالة الطلب.
- فتح واتساب للعميل.
- طباعة الطلب.

حالات الطلب:
- جديد
- تم استلام الطلب
- جاري التجهيز
- خرج للتوصيل
- تم التسليم
- ملغي

-------------

- يتم رفع صور:
  - المنتجات
  - الأقسام
  - المحلات
  - اللوجو/الإعدادات عند الحاجة
- يوجد preview قبل الحفظ.
- يوجد fallback image إذا لم توجد صورة.
- uploadService يتحقق من نوع وحجم الملف.


10. ملاحظات مهمة لأي تطوير لاحق
--------------------------------
- لا تضف تسجيل دخول للعملاء.
- لا تضف دفع أونلاين الآن.
- لا تجعل واجهة العميل معقدة.
- أي تطوير جديد في الأدمن يجب ألا يكسر بساطة واجهة العميل.
- عند تغيير Schema يجب تحديث supabase/schema.sql و README إذا كان التغيير يحتاج تشغيل SQL.
- أي تحديث للمنتجات أو الأقسام يجب أن ينعكس على العميل عبر realtime أو refresh بعد CRUD.
- يفضل أن يكون تطوير الأدمن القادم حول:
  - Bulk price update لعدة منتجات مرة واحدة.
  - استيراد منتجات من CSV.
  - سجل تغييرات للأسعار.
  - صلاحيات أدمن متعددة.
  - تنبيه واضح للمنتجات غير المتوفرة أو بدون صورة.


آخر تحسين تم
-------------
- تحسين صفحة الأقسام بإدارة حالة "قريبًا" وفلاتر وإحصائيات وأزرار سريعة.
- تحسين صفحة المنتجات بإضافة تحديث سريع للأسعار والكمية والتوفر والعروض من نفس كارت المنتج.


آخر تطوير قوي لصفحة المنتجات
-----------------------------
تمت إعادة بناء صفحة /admin/products لتكون مناسبة للاستخدام اليومي السريع من صاحب المشروع أو موظف غير تقني.

ما تم إضافته:
- Top Summary Cards تعرض:
  - إجمالي المنتجات
  - المنتجات المتاحة
  - المنتجات غير المتاحة
  - المنتجات عليها عروض
  - منتجات بدون صورة
  - منتجات سعرها ناقص أو غير مكتمل
- Control Bar يحتوي على:
  - بحث سريع باسم المنتج أو القسم أو المحل أو السعر أو الكلمات المفتاحية
  - فلتر القسم
  - فلتر المحل
  - فلتر الحالة: الكل / متاح / خلصان
  - فلتر العروض
  - Quick filters:
    - عروض فقط
    - غير متاح
    - بدون صورة
    - منتجات بلدي
    - مميزة
    - تحديث اليوم
    - لم تحدث من 30 يوم
    - بيانات ناقصة
- Desktop Table احترافي للمنتجات يحتوي:
  - صورة
  - اسم المنتج
  - القسم
  - المحل
  - السعر
  - السعر القديم
  - الكمية
  - الحالة
  - عرض
  - بلدي
  - آخر تحديث
  - إجراءات
- Mobile Cards مخصصة للموبايل بدل جدول مزدحم.
- Inline Price Editing:
  - الضغط على السعر يحوله إلى input صغير.
  - Enter أو زر الحفظ يحدث السعر مباشرة.
  - يدعم السعر الحالي، السعر القديم، والكمية.
  - تحديث الحالة والعرض والبلدي يتم عبر switches مباشرة.
- Quick Add Product:
  - اسم المنتج
  - القسم
  - السعر
  - الوحدة
  - متاح / خلصان
  - صورة اختيارية
  - أزرار: حفظ المنتج، إضافة منتج آخر، فتح النموذج الكامل
- Full Product Form منظم في Sections:
  - البيانات الأساسية
  - السعر والكمية
  - التصنيف والظهور
  - البحث والكلمات المفتاحية
  - الصورة
- Duplicate Product:
  - زر "نسخ" يفتح نموذج إضافة منتج جديد بنفس بيانات المنتج.
  - الاسم يصبح "نسخة من [اسم المنتج]".
- Bulk Price Update:
  - اختيار متعدد للمنتجات.
  - زيادة بنسبة.
  - تخفيض بنسبة.
  - إضافة مبلغ ثابت.
  - خصم مبلغ ثابت.
  - جعل السعر القديم = السعر الحالي.
  - جعل المنتجات عروض.
  - إلغاء العروض.
  - Preview قبل تأكيد التحديث.
  - Validation يمنع السعر أقل من صفر.
- Export CSV:
  - تصدير المنتجات الظاهرة حاليًا.
  - الأعمدة:
    id, name, category, price, old_price, unit, stock_quantity, available, is_offer
- Import CSV:
  - أصبح يعمل لتحديث الأسعار والكمية والتوفر للمنتجات الموجودة.
  - الأدمن يرفع ملف CSV يحتوي:
    id, price, old_price, stock_quantity, available
  - النظام يقرأ الملف ويعرض Preview قبل الحفظ.
  - يوضح أخطاء الصفوف مثل:
    - id غير موجود
    - المنتج غير موجود
    - سعر غير صحيح
    - كمية غير صحيحة
    - حالة available غير صحيحة
  - لا يحذف منتجات ولا ينشئ منتجات جديدة.
  - يحدث المنتجات الموجودة فقط عن طريق ID.
- Product Image UX:
  - Drag and drop.
  - Preview واضح.
  - تغيير الصورة.
  - حذف الصورة.
  - Badge للمنتج بدون صورة.
- Performance:
  - Search debounce.
  - Pagination.
  - Lazy loading للصور.
  - Optimistic update عند تعديل السعر أو الحالة.


تحسينات سرعة الاستخدام اليومية
-------------------------------
تمت إضافة تحسينات إضافية إلى /admin/products لتقليل الضغطات وتسريع مراجعة الأسعار اليومية:

- وضع تحديث الأسعار:
  - زر "وضع تحديث الأسعار" يحول الصفحة إلى جدول سريع لتعديل:
    - السعر الحالي
    - السعر القديم
    - الكمية
    - متاح / خلصان
    - عرض / ليس عرض
  - يمكن التنقل بين الخانات بالـ Tab.
  - Enter يحفظ الصف.
  - يظهر Badge "تم التعديل" للصفوف المعدلة.
  - يوجد زر "حفظ كل التعديلات".
  - يوجد زر "إلغاء التعديلات".

- حماية التغييرات غير المحفوظة:
  - يظهر شريط تنبيه: "لديك تغييرات غير محفوظة".
  - أزرار: حفظ التغييرات / تجاهل التغييرات.
  - عند محاولة إغلاق الصفحة تظهر حماية browser قبل الخروج.

- تحسين Quick Add:
  - بعد إضافة المنتج تظهر اختيارات:
    - إضافة منتج آخر
    - تعديل التفاصيل الكاملة
    - عرض المنتج في صفحة العملاء
  - عند اختيار القسم يتم اقتراح الوحدة المناسبة.
  - يوجد قوالب:
    - منتج بقالة
    - منتج خضار
    - منتج منظفات
    - منتج عرض
    - منتج بلدي
  - يوجد كشف تكرار بسيط يعرض منتجات متشابهة مع زر فتح المنتج الموجود وزر إضافة على أي حال.

- مراجعة الأسعار اليومية:
  - صندوق أعلى الصفحة يعرض:
    - منتجات لم يتم تحديثها منذ 30 يوم
    - عروض بدون سعر قديم
    - منتجات سعرها ناقص
    - منتجات غير متوفرة
    - منتجات بدون صورة
    - منتجات كمية قليلة
  - كل بند يطبق فلتر مناسب عند الضغط عليه.

- المخزون والتوفر:
  - فلتر سريع باسم "المخزون والتوفر".
  - المنتجات ذات stock_quantity <= 3 تظهر Badge "كمية قليلة".
  - المنتجات ذات stock_quantity = 0 تظهر Badge "خلصان".

- تحسين CSV:
  - Import CSV يعرض Summary:
    - صفوف مقروءة
    - سيتم تحديثها
    - بها أخطاء
    - ID غير موجود
  - Preview يعرض:
    - اسم المنتج الحالي
    - السعر الحالي
    - السعر الجديد
    - الكمية الحالية
    - الكمية الجديدة
    - الحالة الحالية
    - الحالة الجديدة
  - الصفوف الخطأ يتم تخطيها عند التأكيد.
  - رسالة النتيجة تعرض عدد المنتجات المحدثة وعدد الصفوف الفاشلة.

- تحسين Export CSV:
  - أصبح التصدير من Modal بخيارات:
    - كل المنتجات
    - المنتجات الظاهرة بعد الفلترة
    - منتجات قسم محدد
    - المنتجات غير المتوفرة
    - المنتجات بدون صورة
    - العروض فقط

- Keyboard Shortcuts:
  - Ctrl + K للبحث.
  - Ctrl + N لإضافة منتج سريع.
  - Ctrl + S لحفظ التغييرات في وضع تحديث الأسعار.
  - Esc لإغلاق المودالات.

- Safety:
  - لو المنتج عليه طلبات سابقة، تظهر رسالة تحذير قبل الحذف وتقترح إيقاف المنتج بدل الحذف.
  - السعر لا يكون أقل من 0.
  - السعر القديم في حالة العرض يجب أن يكون أكبر من السعر الحالي إذا تم إدخاله.
  - المنتج العرض بدون سعر قديم يظهر له warning.
